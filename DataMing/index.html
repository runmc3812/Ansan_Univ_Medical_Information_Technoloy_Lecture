1. 데이터마이닝의 개념 및 필요성

정의 및 목적

  • 정의: 대량의 데이터 집합에서 유용한 정보, 패턴, 추세를 자동/반자동으로 발견하는 과정. (데이터 광산에서 금 캐기 비유)
  • 필요성: 보건의료 데이터 폭증, 증거 기반 의학(EBM) 및 예측의 중요성 증대.
  • 경쟁력: 기술과 자본이 비슷한 상황에서 정보력의 차이가 곧 경쟁력의 차이.

기술 진화 및 비교

DB 기술은 수집에서 예측(지식발견)으로 진화했습니다.

  • 데이터 진화: 데이터 수집 → DB 관리 → DW (통합/분석) → DM (예측/지식발견).
  • 주요 처리 방식: DB (OLTP) / DW (OLAP) / DM (데이터마이닝 기법 적용).

DB vs DW vs Data Mining 비교

구분 데이터베이스 (운영 DB) 데이터웨어하우스 (DW) 데이터마이닝 (DM)
데이터 용도 저장, 삭제, 수정, 검색 데이터의 통합 및 정제 (읽기 전용) 분류와 예측에 사용
주요 업무 일반적인 업무 처리 (e.g., 진료 접수) 대용량 정보 분석 (e.g., 월간 통계) 지식 발견 (e.g., 질병 예측)

2. 주요 기능 및 학습 방법

주요 기능 (5가지)

  • 분류: 데이터를 정해진 그룹으로 나눔 (e.g., 고위험군 / 저위험군).
  • 예측과 추정: 미래 값이나 확률을 예측 (e.g., 예상 재원일수, 질병 발병 확률).
  • 연관성 규칙: 항목 간의 동시 발생 패턴을 찾음 (e.g., 당뇨 환자가 고혈압을 동반할 확률).
  • 군집: 유사한 특성을 가진 데이터들을 그룹(Cluster)으로 묶음.
  • 데이터 축소: 핵심 요인만 식별하여 데이터 차원을 줄임.

학습 방법 (2가지)

  • 1. 지도 학습 (Supervised Learning):

    정답(Label)이 있는 데이터로 학습, 출력변수 존재.

    → 주요 기능: 분류, 예측 (e.g., 암 확진 여부 기반 예측).

  • 2. 비지도 학습 (Unsupervised Learning):

    정답 없이 스스로 데이터의 구조/패턴을 찾게 함, 출력변수 없음.

    → 주요 기능: 군집, 연관성 규칙 (e.g., 내원 패턴 기반 환자 그룹화).

3. 데이터마이닝 수행 절차 및 핵심 기법

수행 절차 (6단계 흐름)

목적 확인 → 데이터 수집 → 전처리(70~80% 소요) → 변환/축소 → DM 수행 → 해석/평가/활용

  • 전처리/변환: 결측치/이상치 처리, 분석에 적합한 형태로 데이터 가공.
  • 평가/활용: 모델 정확도 검증 (Accuracy, AUROC), 현업 적용(Deployment).

4. 보건의료 분야 활용 사례 (인공지능)

병원 경영 / 행정

  • 병원 수익 영향 요인 분석
  • 미수금 발생 확률이 높은 환자 예측
  • 진료비 삭감 내역 분석 및 원인 규명
  • 환자 특성 분석 (군집분석, CRM)

임상 연구 / 진료 (AI)

  • 의료 영상 판독: X-ray, CT 등의 영상 분석 및 정상/비정상 분류
  • 치료법 도출: 환자 데이터 기반 진단 및 치료 권고 (e.g., IBM Watson Health)
  • 질병 관계 예측: 질병 간의 연관성 및 발병 확률 예측

질병 발병 및 중증도 예측

  • 암 재발 환자 예측 (신경망)
  • 환자 생체신호를 분석하여 심정지 예측
  • 개인 건강검진 기록 기반 주요 질환 위험도 예측 (e.g., 당뇨, 치매, 암)
