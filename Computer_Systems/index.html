0) 개요

컴퓨터시스템은 하드웨어 위에 소프트웨어가 올라가 이진수 데이터를 처리하는 전자장치입니다. 저장장치(RAM/ROM/보조기억장치)와 CPU, I/O가 유기적으로 동작합니다.

  • 데이터 표현: 이진수(0/1)
  • 실행 흐름: 보조기억장치 → RAM 적재 → CPU 실행 → 결과 저장

1) 컴퓨터란? (What)

정의 하드웨어 위에 소프트웨어가 올라가 데이터를 이진수로 저장·처리하는 장치입니다. 기억장치에 저장된 내용을 CPU가 불러와 처리하고, 결과를 출력합니다.

2) 하드웨어(HW)

2-1. 구성요소

  • CPU(ALU·레지스터·제어장치·버스)
  • 기억장치(RAM/ROM/보조기억장치)
  • I/O(입출력장치: 키보드·마우스·모니터 등)

2-2. CPU 핵심

  • ALU: 산술·논리 연산
  • 레지스터: 초고속 임시 저장
  • 제어장치: 명령 해석/제어 신호
  • 버스: 주소/데이터/제어 신호 전달

2-3. 기억장치/계층

  • RAM: 주기억·휘발성
  • ROM/Flash: 비휘발
  • 보조기억: HDD/SSD 등
  • 계층: 레지스터/캐시 → RAM → SSD/HDD (속도↓ 용량↑)

2-4. 입출력장치

  • 입력: 키보드·마우스·스캐너·마이크
  • 출력: 모니터·프린터·스피커

3) 소프트웨어(SW)

3-1. 분류

  • 시스템 SW: 운영체제, 드라이버, 컴파일러/링커
  • 응용 SW: 업무/의료 등 응용 프로그램
  • 상용/프리/셰어, 패키지/프로토타입 등

3-2. 운영체제(OS)

  • 역할: 자원관리(CPU/메모리/장치), 프로세스·파일 관리
  • 커널: 핵심 기능, 쉘: 명령 해석기
  • 예) Windows, macOS, Linux, Unix, iOS 등

3-3. 컴파일러·링커

  • 컴파일러: 고급언어 → 목적코드
  • 링커: 목적파일+라이브러리 결합 → 실행파일
  • 흐름: 번역(컴파일) → 조립(링크)

3-4. 의료 SW

  • 업무 SW 중 일부는 의료기기 SW 규제 대상
  • 안전성/유효성, 품질관리 요구

4) 데이터 통신/네트워크

4-1. 클라이언트/서버

  • 클라이언트: 요청·표시
  • 서버: 저장·처리
  • 클라우드/분산으로 경계 유연화

4-2. 프로토콜/OSI

  • 프로토콜: HTTP, TLS, TCP/UDP, IP 등
  • OSI 7계층: 응용–표현–세션–전송–네트워크–데이터링크–물리

4-3. 네트워크 기본

  • 전송방식: 단방향/반이중/전이중
  • 거리: LAN/WAN · 토폴로지: 스타/버스/링/메시/트리
  • 전송: 직렬/병렬 · 동기/비동기

4-4. 네트워크 장비

  • NIC, 허브, 스위치(L2), 라우터(L3)
  • AP(무선), 모뎀

5) 디지털 기초

아날로그 vs 디지털

연속값 vs 이산값(0/1). 디지털 회로는 LOW/HIGH로 처리.

수 표현

2진·16진, 비트(최소 단위) · 바이트(8비트)

버스 폭·클럭

버스 폭(표현·전달량), 클럭(동기화, Hz↑ → 간격↓)

6) 한 장 체크리스트

  • CPU의 ALU/레지스터/제어장치/버스 설명
  • RAM·ROM·Flash·보조기억 차이
  • 커널·쉘·컴파일러·링커 역할
  • OSI 7계층 순서
  • 비트·바이트·버스 폭·클럭 의미

부록) 병원정보시스템(예시)

클라이언트–서버 구조 위에 OCS/EMR/PACS 등이 올라가며, 라우터/스위칭 허브로 네트워크를 구성합니다.
